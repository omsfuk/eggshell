# eggshell

伪花生壳。可以让你借助于一台具有公网ip的服务器，实现局域网穿透。

典型应用场景：作为一个后端，当你和前端集成时，每次出问题，都要到服务器查日志，然后回到本地修bug，然后更新服务器，来来回回很麻烦。
此时每每会想倘若自己的机器是公网ip就好了。现在，通过eggshell，使得这成为了可能。

eggshell其实是一个代理，不过这个代理特殊的地方在于：它是用于局域网和公网代理的。即通过一个公网服务器，将请求转发到某一个局域网内。
通过这，**前端从此就可以直连本地debug。**

eggshell理论上适应于所有的代理，不局限于HTTP。
但是经过测试，最起码我的世界代理就没有走通。。当然，HTTP是可行的

配置方法也比较简单，且提供了web接口，一个良好的gui。
在服务器启动eggshell，访问localhost:8081，则会出现控制界面，然后选择“服务器”，默认设置启动。

然后在服务端启动eggshell，访问localhost:8081，选择客户端，默认设置启动。

参数名称都很明确，一眼就能看出来它的目的。
